# Code Of Conduct

Memory Genome Engine should stay practical, technical, and respectful.

## Expected Behavior

- Keep discussion focused on code, docs, security, benchmarks, and product quality.
- Be direct without personal attacks.
- Respect privacy: do not post secrets, passphrases, private memory stores, or sensitive corpus data.
- Assume maintainers may reject changes that increase complexity without clear value.

## Unacceptable Behavior

- Harassment, threats, hate speech, or targeted personal attacks.
- Publishing private security details before maintainers have a chance to respond.
- Sharing another person's private data, credentials, or memory store contents.
- Repeatedly pushing large architecture changes after maintainers have asked for a design discussion first.

## Enforcement

Maintainers may edit or remove comments, close issues, reject pull requests, or restrict participation when behavior harms the project. Security-sensitive issues should follow [SECURITY.md](SECURITY.md).
